# FIRST LINE site

Минимальный статический лендинг FIRST LINE на чистом HTML/CSS/JS/PHP без сборщика, npm, React, Next.js, Vite и внешних зависимостей.

## Структура проекта

```text
/
├── index.html
├── privacy.html
├── send.php
├── README.md
├── AGENTS.md
└── assets/
    └── images/
        └── .gitkeep
```

## Настройка формы

Перед загрузкой на хостинг откройте `send.php` и замените значения переменных на реальные адреса:

```php
$to = "ВАША_ПОЧТА@yandex.ru";
$from = "no-reply@ВАШ-ДОМЕН.ru";
```

- `$to` — адрес Яндекс.Почты, куда будут приходить заявки.
- `$from` — почтовый адрес на вашем домене. На хостинге Beget для стабильной доставки лучше использовать адрес, созданный для этого домена.

Форма в `index.html` отправляет POST-запрос в `send.php`. В обработчике есть honeypot-поле, базовая серверная проверка телефона и отправка письма через PHP `mail()`.

## Изображения объектов

Папка `assets/images/` сохранена в репозитории через текстовый файл `.gitkeep`. Файлы `object-1.jpg` — `object-5.jpg` в PR не добавляются: реальные фотографии объектов нужно загрузить вручную позже с теми же именами, чтобы не менять HTML.

## Загрузка на Beget

1. Войдите в панель управления Beget или подключитесь по FTP/SFTP.
2. Откройте папку сайта `public_html` нужного домена.
3. Загрузите в `public_html` все файлы и папки из корня проекта:
   - `index.html`
   - `privacy.html`
   - `send.php`
   - `README.md` (необязательно для работы сайта)
   - `AGENTS.md` (необязательно для работы сайта)
   - папку `assets/` целиком
4. Если фотографии объектов уже готовы, загрузите их вручную в `public_html/assets/images/` с именами `object-1.jpg` — `object-5.jpg`.
5. Откройте домен в браузере и проверьте якорные ссылки, карточки объектов и отправку формы.

## Ограничения проекта

- Не добавлять `package.json`, npm, сборщики, React, Next.js, Vite или другие фреймворки.
- Не подключать внешние CDN и Google Fonts.
- Сохранять утверждённый дизайн и тексты лендинга без необходимости изменений.
- Форма обратной связи должна использовать `send.php`.
- Не коммитить изображения, PDF или другие бинарные файлы; папка `assets/images/` хранится через `.gitkeep`.
